# UI Principles

## Audience
Beginner robotics learners who may be more comfortable with systems and mechanics than software architecture.

## Goals
- make ROS architecture feel understandable
- reduce intimidation
- show cause and effect clearly
- support exploration without clutter

## Priorities
1. visual hierarchy
2. legibility
3. clear active states
4. clear labels
5. low cognitive load

## Avoid
- crowded panels
- tiny labels
- controls without clear teaching value
- advanced terminology without framing
- motion that harms readability

## Preferred changes
- spacing improvements
- clearer grouping
- better labels
- stronger selected / active / recent states
- calmer layouts

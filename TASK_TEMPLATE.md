# Task Template

## Task type
bugfix / UI polish / refactor / feature / docs

## Goal
What should be better after this task?

## Problem
What is happening now?

## Expected behavior
What should happen instead?

## Context
Relevant files, components, routes, commands, or user flow.

## Constraints
What must not break?
What should stay unchanged?

## Acceptance criteria
Concrete signs the task succeeded.

## Validation
Exact commands to run and exact behavior to verify.

## Deliverable format
Return:
1. diagnosis or plan
2. files changed
3. key behavior change
4. validation performed
5. risks / follow-ups
